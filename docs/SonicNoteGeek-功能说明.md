# SonicNoteGeek 插件功能说明

> **版本**：v1.0.0  
> **适用平台**：Obsidian Desktop（Windows / macOS / Linux）  
> **核心功能**：音频转写 + 说话人识别 + AI 智能总结  

---

## 一、插件概述

SonicNoteGeek 是一个 Obsidian 笔记插件，能将音频文件自动转为带说话人标注的文字稿，并调用大语言模型（LLM）生成结构化总结，最终输出到当前 Markdown 文档中。

### 核心流程

**本地转写流程：**

```
音频 URL/文件 → 下载到 Vault → 语音转写 (ASR) → 说话人分离 →
声纹匹配 (可选) → AI 总结 (LLM) → 写入文档
```

**云端同步流程：**

```
妙记 App 录音 → 云端自动转写/AI总结 → 插件一键同步 →
转录原文 + AI 总结自动写入本地 Obsidian 文档
```

## 四、初始配置（首次使用）

### 4.1 配置 ASR 语音转写模型

1. 点击面板右上角 **☰ 菜单按钮**
2. 选择 **ASR 配置**
3. 在弹出的配置窗口中：
   - 点击 **+ 添加模型**
   - 选择服务商（如 OpenAI Whisper、火山引擎、阿里云、讯飞等）
   - 填写 API Key、API URL、模型名称
   - 点击保存
4. 在模型列表中，**点击某个模型** 将其设为当前激活模型（显示绿色边框）

### 4.2 配置 LLM 大语言模型

1. 点击 ☰ 菜单 → **LLM 配置**
2. 点击 **+ 添加模型**
3. 选择服务商（Anthropic Claude、OpenAI GPT、智谱 GLM、DeepSeek、MiniMax 等）
4. 填写 API Key、API URL、模型名称
5. 点击模型激活

### 4.3 配置热词（可选）

1. ☰ 菜单 → **热词管理**
2. 输入行业术语、专有名词，点击添加
3. 支持热词库管理，目前火山引擎/讯飞/腾讯云支持热词

### 4.4 配置声纹识别（可选）

1. ☰ 菜单 → **声纹识别**
2. 填写声纹识别服务地址（如 CAM++ 服务 `http://localhost:8100`）
3. 在声纹库中添加说话人：姓名 + 3~10秒的清晰单人音频样本
4. 开启「自动声纹识别」即可在转写时自动匹配人名

## 五、开始转写总结（完整流程）

### 5.1 准备音频

音频来源支持两种方式：

| 方式 | 说明 |
|------|------|
| 公网 HTTP URL | 在文档中粘贴 MP3 链接，面板自动检测并列出 |
| 手动输入 | 在音频链接区域手动输入 URL |

### 5.2 操作步骤

1. **在 Obsidian 中打开一个 Markdown 文档**（转写结果会写入此文档）
2. **确保文档中有音频链接**，或在面板中手动输入
3. **勾选要处理的音频文件**（支持多选）
4. **选择总结模板**（点击模板卡片，当前选中会有绿色边框）
5. **点击「开始转写总结」按钮**

### 5.3 处理进度

按钮文字会依次显示：

```
处理中... → 音频下载... → 语音转写... → 分析总结... → 文档写入...
```

进度条四个圆点依次点亮，对应四个阶段：

| 阶段 | 图标 | 说明 |
|------|------|------|
| 下载 | ● | 从公网下载 MP3 到 Vault 本地 |
| 转写 | ● | 调用 ASR 引擎转为文字稿 |
| 总结 | ● | 调用 LLM 生成结构化总结 |
| 写入 | ● | 将结果写入当前 MD 文档 |

### 5.4 输出文档格式

处理完成后，文档末尾会追加如下内容：

```markdown
## AI 智能总结
（LLM 生成的会议纪要/课堂笔记等...）

---

## 关键词
`AI`  `语音识别`  `大模型`

## 行动项
- [ ] 完成ASR接口对接
- [ ] 下周一前提交测试报告

---

## 转录信息
- **时长**：15分30秒
- **语言**：zh
- **ASR 引擎**：OpenAI Whisper
- **说话人数**：3
- **生成模板**：商务会议
- **生成时间**：2026-06-20
- **原始音频**：https://example.com/audio.mp3

---

## 转录原文

**[00:00:05] 说话人1：** 今天我们讨论一下新项目的排期...

**[00:00:12] 说话人2：** 我觉得可以先从需求评审开始...

**[00:00:20] 说话人3：** 同意，我这边的资源下周可以到位...
```

## 六、操作按钮说明

### 6.1 重新转写

- **用途**：保留原文档前面的总结内容，仅重新执行 ASR 转写 + 说话人分离
- **结果**：旧的「转录原文」部分被新的替换，AI 总结不变
- **适用场景**：更换 ASR 引擎后想重新生成逐字稿；或调整声纹库后人名匹配不准确

### 6.2 重新总结

- **用途**：保留转录原文不变，重新调用 LLM 生成总结
- **点击后弹出模板选择器**，可选择不同模板重新生成
- **结果**：在原「转录信息」之前插入新的「重新总结」章节
- **适用场景**：对当前总结不满意，换模板或换 LLM 模型重试

### 6.3 人声勘正

- **用途**：将转录原文中的"说话人1、说话人2、说话人3"自动匹配为声纹库中的真实姓名
- **前提**：已配置声纹识别服务、声纹库中有带音频样本的说话人
- **结果**：文档中所有 `说话人N` 替换为匹配到的真实姓名
- **适用场景**：首次转写时未开自动声纹识别，事后想补充人名标注

### 6.4 声纹采样

- **用途**：从转录原文中提取某个说话人的音频片段，保存为声纹样本
- **操作步骤**：
  1. 点击「声纹采样」按钮
  2. 弹窗列出所有逐字稿中的说话人及其文本
  3. 每位说话人可选 3 个候选音频片段（基于最长段落）
  4. 点击 ▶ 播放按钮试听
  5. 点击 🔄 切换候选片段
  6. 满意的片段点击「添加声纹库」，输入姓名后保存
  7. 也可点击底部「一键全部采样」批量提取

### 6.5 刷新按钮

- **用途**：重新检测当前文档中的音频链接、转录原文、AI 总结
- **按钮启用条件**：
  - **重新转写**：检测到音频文件
  - **重新总结**：检测到转录原文
  - **人声勘正**：检测到转录原文 + 音频文件
  - **声纹采样**：检测到转录原文 + 音频文件

## 七、云端文件同步

### 7.1 功能概述

插件支持从 **SonicNote 妙记 App** 云端同步录音文件到本地 Obsidian Vault。你在手机上用妙记 App 录制的音频，App 会自动完成云端转写和 AI 总结；回到电脑上，一键即可将转录原文和 AI 总结同步到 Obsidian 文档。

### 7.2 前置配置

1. 打开妙记 App → 我的 → API Key 管理 → 创建 Key（格式：`sk-xxx`）
2. 在插件面板中点击左上角 **☁️ 文件同步** 按钮
3. 首次使用会弹出设置窗口，填写：
   - **API Key**：上一步获取的 Key
   - **服务器地址**：`https://ainote.easylinkin.com:18048/prod-api`（默认）
   - **同步目录**：Obsidian Vault 中的存放路径（默认 `SonicNoteSync/`）
   - **Frontmatter 字段**：文档 YAML 头部记录的元数据字段名

### 7.3 同步流程

```
妙记 App 录音 → 云端 ASR 转写 → 云端 AI 总结
                    ↓
        插件点击「文件同步」
                    ↓
    ├─ 从云端拉取录音列表
    ├─ 对比本地 SonicNoteSync/ 目录
    ├─ 下载新增/更新的录音
    └─ 为每条录音生成 Markdown 文档:
         ├─ YAML frontmatter（audioId、时长、创建时间、音频URL）
         ├─ AI 智能总结（如果有）
         ├─ 转录原文（含说话人标注）
         └─ 音频文件下载到 SonicNoteSync/Audio/{文档名}/
```

### 7.4 文档输出格式

每条同步的录音在 Vault 中生成一个独立的 Markdown 文件，格式如下：

```markdown
---
audio_id: abc123
title: 项目周会讨论
duration: 1830
created_at: 2026-06-25T10:30:00+08:00
audio_url: https://ainote.easylinkin.com/...
---

## AI 智能总结
（妙记 App 云端生成的 AI 总结）

---

## 转录信息
- **时长**：30分30秒
- **语言**：zh
- **录音ID**：abc123

---

## 转录原文

**[00:00:05] 说话人1：** 今天我们讨论一下项目进展...

**[00:00:12] 说话人2：** 上周的需求评审已经完成了...
```

### 7.5 更新说明

- 插件在同步时检测云端录音的更新时间，自动跳过本地已是最新的文件
- 云端转写或总结新增时，同步按钮旁会显示待同步数量
- 同步目录下的文件遵循 `SonicNoteSync/` 目录结构，与其他音频处理功能共享

### 7.6 同步按钮

工具栏左上角的 **☁️ 文件同步** 按钮：
- 点击触发增量同步
- 同步中按钮禁用
- 完成后显示同步数量提示

## 九、AI 聊天对话

### 8.1 功能说明

面板底部的 AI 聊天区域，可以针对当前转录内容进行问答。

### 8.2 使用方式

1. 至少完成过一次转写（插件会记住最近的转录内容）
2. 在底部输入框中输入问题，如：
   - "这篇会议主要讨论了什么？"
   - "有哪些人提到了预算问题？"
   - "把待办事项整理成表格"
3. 点击「发送」或按 Enter
4. AI 回复会显示在上方对话区

### 8.3 技术说明

- 聊天使用当前激活的 LLM 模型
- 对话上下文包含最近一次转写的完整转录原文
- 支持多轮对话

## 十二、常见问题

### Q: 阿里云等云端 ASR 为什么重新转写失败？

A: 阿里云 DashScope、OpenAI Whisper、火山引擎等云端 ASR 只接受**公网可访问的 HTTP URL**，不支持本地文件。本地文件请使用**讯飞**或**本地 OpenAI 兼容**引擎。

### Q: 转录原文中的"说话人1、说话人2"能自动显示名字吗？

A: 配置声纹识别服务（CAM++）并在声纹库中添加说话人样本后，开启「自动声纹识别」即可。也可事后通过「人声勘正」按钮手动匹配。

### Q: AI 总结是英文怎么办？

A: 在模板的 System Prompt 开头明确写"使用中文输出"。或者选择内置的中文模板（商务会议、课堂总结等）。

### Q: 支持哪些音频格式？

A: MP3、WAV、M4A、FLAC、OGG、AAC、WEBM。

### Q: 处理超大音频会超时吗？

A: 各引擎超时时间不同，最长轮询等待 10 分钟（讯飞/百度/华为云）、阿里云 6 分钟、火山引擎 4 分钟。
